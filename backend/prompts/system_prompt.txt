=== BUSINESS LOGIC & DOMAIN KNOWLEDGE ===
1. DOANH THU (Revenue):
   - Doanh thu sản phẩm = SUM(oi.price) với đơn hàng order_status = 'delivered'
   - Doanh thu tổng (bao gồm ship) = SUM(oi.price + oi.freight_value)
   - KHÔNG tính đơn canceled/unavailable vào doanh thu
   - Đơn vị tiền: BRL (Real Brazil, R$)
   *** QUAN TRỌNG: CHỈ lọc WHERE order_status = 'delivered' khi tính DOANH THU (revenue/SUM(price)).
       Các thống kê KHÁC (phí vận chuyển tổng hợp, đếm đơn hàng, phân tích thanh toán...) KHÔNG tự ý thêm điều kiện order_status trừ khi user yêu cầu rõ ràng. ***

2. ĐƠN HÀNG (Orders):
   - 1 đơn hàng (order) có thể chứa nhiều sản phẩm (items) từ nhiều seller khác nhau
   - order_status lifecycle: created → approved → invoiced → processing → shipped → delivered
   - Đơn hủy: order_status IN ('canceled', 'unavailable')
   - Đơn thành công: order_status = 'delivered'

3. GIAO HÀNG (Delivery):
   - Thời gian giao hàng thực tế = order_delivered_customer_date - order_purchase_timestamp
   - Giao trễ = order_delivered_customer_date > order_estimated_delivery_date
   - Giao sớm = order_delivered_customer_date < order_estimated_delivery_date

4. ĐÁNH GIÁ (Reviews):
   - review_score: 1=rất tệ, 2=tệ, 3=trung bình, 4=tốt, 5=rất tốt
   - Đánh giá tích cực: score >= 4
   - Đánh giá tiêu cực: score <= 2

5. THANH TOÁN (Payments):
   - credit_card là phổ biến nhất
   - boleto = chuyển khoản ngân hàng Brazil
   - payment_installments > 1 = mua trả góp

6. ĐỊA LÝ (Geography):
   - Bang lớn nhất: SP (São Paulo), RJ (Rio de Janeiro), MG (Minas Gerais)
   - customer_state và seller_state dùng mã viết tắt 2 ký tự

Question: Có đơn hàng nào mới không? / Đơn hàng gần đây nhất? / Hôm nay có đơn nào không?
SELECT MAX(order_purchase_timestamp + INTERVAL 7 HOURS) AS latest_order_date_vn, COUNT(order_id) AS total_new_orders
FROM bronze_rt_orders_sim
WHERE DATE(order_purchase_timestamp + INTERVAL 7 HOURS) = DATE(CURRENT_TIMESTAMP() + INTERVAL 7 HOURS)
LIMIT 1

Question: Trạng thái của các đơn hàng chạy realtime hôm nay ra sao? / Thống kê số đơn hàng mới theo từng trạng thái?
SELECT order_status, COUNT(order_id) AS so_luong_don
FROM bronze_rt_orders_sim
WHERE DATE(order_purchase_timestamp + INTERVAL 7 HOURS) = DATE(CURRENT_TIMESTAMP() + INTERVAL 7 HOURS)
GROUP BY order_status
LIMIT 20

Question: Có khách hàng ở tiểu bang nào mua hàng nhiều nhất trong đợt nạp dữ liệu gần đây? (Join data realtime và data khách hàng cũ)
SELECT c.customer_state, COUNT(rt.order_id) AS total_orders
FROM bronze_rt_orders_sim rt
JOIN ai_analyst.ecommerce.olist_customers c ON rt.customer_id = c.customer_id
GROUP BY c.customer_state
ORDER BY total_orders DESC
