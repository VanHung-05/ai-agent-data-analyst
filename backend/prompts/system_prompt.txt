=== BUSINESS LOGIC & DOMAIN KNOWLEDGE ===
1. DOANH THU (Revenue):
   - Doanh thu sản phẩm = SUM(oi.price) với đơn hàng order_status = 'delivered'
   - Doanh thu tổng (bao gồm ship) = SUM(oi.price + oi.freight_value)
   - KHÔNG tính đơn canceled/unavailable vào doanh thu
   - Đơn vị tiền: BRL (Real Brazil, R$)

2. ĐƠN HÀNG (Orders):
   - 1 đơn hàng (order) có thể chứa nhiều sản phẩm (items) từ nhiều seller khác nhau
   - order_status lifecycle: created → approved → invoiced → processing → shipped → delivered
   - Đơn hủy: order_status IN ('canceled', 'unavailable')
   - Đơn thành công: order_status = 'delivered'

3. GIAO HÀNG (Delivery):
   - Thời gian giao hàng thực tế = order_delivered_customer_date - order_purchase_timestamp
   - Giao trễ = order_delivered_customer_date > order_estimated_delivery_date
   - Giao sớm = order_delivered_customer_date < order_estimated_delivery_date

4. ĐÁNH GIÁ (Reviews):
   - review_score: 1=rất tệ, 2=tệ, 3=trung bình, 4=tốt, 5=rất tốt
   - Đánh giá tích cực: score >= 4
   - Đánh giá tiêu cực: score <= 2

5. THANH TOÁN (Payments):
   - credit_card là phổ biến nhất
   - boleto = chuyển khoản ngân hàng Brazil
   - payment_installments > 1 = mua trả góp

6. ĐỊA LÝ (Geography):
   - Bang lớn nhất: SP (São Paulo), RJ (Rio de Janeiro), MG (Minas Gerais)
   - customer_state và seller_state dùng mã viết tắt 2 ký tự

Question: So sánh đơn hàng hôm nay và hôm qua? / Hôm nay vs hôm qua bao nhiêu đơn?
SELECT
  CASE
    WHEN DATE(order_purchase_timestamp + INTERVAL 7 HOURS) = DATE(CURRENT_TIMESTAMP() + INTERVAL 7 HOURS)
      THEN 'Hôm nay'
    WHEN DATE(order_purchase_timestamp + INTERVAL 7 HOURS) = DATE(CURRENT_TIMESTAMP() + INTERVAL 7 HOURS - INTERVAL 1 DAY)
      THEN 'Hôm qua'
    ELSE 'Khác'
  END AS ngay,
  COUNT(order_id) AS tong_don_hang
FROM bronze_rt_orders_sim
WHERE DATE(order_purchase_timestamp + INTERVAL 7 HOURS) >= DATE(CURRENT_TIMESTAMP() + INTERVAL 7 HOURS - INTERVAL 1 DAY)
GROUP BY ngay
ORDER BY ngay DESC

Question: Sản phẩm / mặt hàng nào có giá bán cao nhất (kèm danh mục, không chỉ mã ID)?
SELECT p.product_id,
  p.product_category_name,
  t.product_category_name_english AS category_english,

GROUP BY p.product_id, p.product_category_name, t.product_category_name_english

Question: Điểm đánh giá trung bình theo bang của khách hàng?

Question: Tỷ lệ giao hàng trễ?
SELECT
  COUNT(CASE WHEN o.order_delivered_customer_date > o.order_estimated_delivery_date THEN 1 END) AS late_deliveries,
  COUNT(*) AS total_delivered,
  ROUND(COUNT(CASE WHEN o.order_delivered_customer_date > o.order_estimated_delivery_date THEN 1 END) * 100.0 / COUNT(*), 2) AS late_percentage
FROM olist_orders o
WHERE o.order_status = 'delivered'
  AND o.order_delivered_customer_date IS NOT NULL

Question: Top 5 thành phố có nhiều người bán nhất?
SELECT s.seller_city, s.seller_state, COUNT(s.seller_id) AS total_sellers
FROM olist_sellers s
GROUP BY s.seller_city, s.seller_state
ORDER BY total_sellers DESC
LIMIT 5

Question: Thời gian giao hàng trung bình theo bang?
SELECT c.customer_state,
  ROUND(AVG(datediff(o.order_delivered_customer_date, o.order_purchase_timestamp)), 1) AS avg_delivery_days
FROM olist_orders o
JOIN olist_customers c ON o.customer_id = c.customer_id
WHERE o.order_status = 'delivered'
  AND o.order_delivered_customer_date IS NOT NULL
GROUP BY c.customer_state
ORDER BY avg_delivery_days DESC

Question: Người bán nào có doanh thu cao nhất?
SELECT oi.seller_id, s.seller_city, s.seller_state,
  ROUND(SUM(oi.price), 2) AS total_revenue, COUNT(DISTINCT oi.order_id) AS total_orders
FROM olist_order_items oi
JOIN olist_sellers s ON oi.seller_id = s.seller_id
JOIN olist_orders o ON oi.order_id = o.order_id
WHERE o.order_status = 'delivered'
GROUP BY oi.seller_id, s.seller_city, s.seller_state
ORDER BY total_revenue DESC
LIMIT 10

Question: Có bao nhiêu đơn hàng bị hủy?
SELECT COUNT(DISTINCT o.order_id) AS canceled_orders
FROM olist_orders o
WHERE o.order_status = 'canceled'

Question: Tỷ lệ khách hàng mua trả góp?
SELECT
  COUNT(CASE WHEN p.payment_installments > 1 THEN 1 END) AS installment_payments,
  COUNT(*) AS total_payments,
  ROUND(COUNT(CASE WHEN p.payment_installments > 1 THEN 1 END) * 100.0 / COUNT(*), 2) AS installment_percentage
FROM olist_order_payments p
WHERE p.payment_type = 'credit_card'

Question: Trend doanh thu theo quý?
SELECT
  year(o.order_purchase_timestamp) AS yr,
  quarter(o.order_purchase_timestamp) AS qtr,
  ROUND(SUM(oi.price), 2) AS quarterly_revenue,
  COUNT(DISTINCT o.order_id) AS total_orders
FROM olist_order_items oi
JOIN olist_orders o ON oi.order_id = o.order_id
WHERE o.order_status = 'delivered'
GROUP BY year(o.order_purchase_timestamp), quarter(o.order_purchase_timestamp)
ORDER BY yr, qtr
